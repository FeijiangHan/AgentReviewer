# Qwen2.5VL 3B 文档版面检测任务：输入 / 处理 / 输出全流程说明

本文对你给出的任务配置进行结构化说明，重点解释 **输入（Input）—处理（Process）—输出（Output）** 三个环节，并明确每一步如何与最终指标 `mAP50` 对齐。

---

## 1. 输入（Input）

这一环节的目标是把「模型推理需要的信息」和「评测需要的真值信息」准备好。

### 1.1 模型与权重输入

- **模型类型**：Qwen2.5VL 3B（视觉语言模型，用于根据图像 + prompt 输出版面区域）
- **checkpoint 来源**：
  - `MODEL_BASE_ID = "ChaseHan/Latex2Layout-2000-sync"`（Hugging Face）
- **任务形式**：给模型输入页面图像和固定 prompt，让模型输出检测框与类别。

### 1.2 图像输入

- **输入图像目标尺寸**：
  - `TARGET_SIZE = (924, 1204)`
- 含义：所有测试页在送入模型前，需统一缩放/对齐到该尺寸（或采用等效的 letterbox 策略并维护坐标映射）。

### 1.3 文本 prompt 输入

- 使用给定 `DEFAULT_PROMPT`，核心要求模型检测如下区域：
  - `"title", "abstract", "heading", "footnote", "figure", "figure caption", "table", "table caption", "math", "text"`
- 返回格式被约束为 JSON：
  - 每个元素含 `bbox_2d`、`label`、`order`。

> 说明：虽然 prompt 包含 `text`，但你当前评测要求只保留 9 类（不含 `text`）。因此后续会在预测和标注两侧统一过滤，确保指标计算口径一致。

### 1.4 数据集输入

- 数据集：DocLayNet 测试集（或你选择的等价 layout 数据集测试集）。
- 样本规模：**只加载 1K 张页面**。
- 真值标签（GT）处理：从官方标签中过滤，仅保留以下 9 类：
  1. title
  2. abstract
  3. heading
  4. footnote
  5. figure
  6. figure caption
  7. table
  8. table caption
  9. math

---

## 2. 处理（Process）

处理环节分为：数据采样 → 预处理 → 推理 → 后处理 → 指标计算。

### 2.1 测试集采样与标签映射

1. 从测试集读入样本列表。
2. 取前 1,000 页（或固定随机种子下采样 1,000 页，确保可复现）。
3. 将数据集原始类别映射到目标类别空间。
4. 过滤 GT：只保留 9 个目标类别标注框；其他类别直接丢弃。

**关键点**：预测类别空间与 GT 类别空间必须完全一致，否则 mAP50 会被错误拉低。

### 2.2 图像预处理

对每页图像执行：

- resize 到 `TARGET_SIZE = (924, 1204)`（或等效可逆方案）；
- 若保持长宽比 + padding，则记录缩放比例和 pad 偏移，便于把预测框映射回评测坐标；
- 构造模型输入张量/多模态输入格式。

### 2.3 模型推理（Inference）

每页调用一次模型生成：

- 输入：`<image> + DEFAULT_PROMPT`
- 输出：包含若干对象的 JSON 字符串，字段一般为：
  - `bbox_2d: [x1, y1, x2, y2]`
  - `label: "..."`
  - `order: "..."`

### 2.4 预测后处理

从模型文本输出到可评测检测结果，通常需要以下步骤：

1. **解析 JSON**：提取 bbox 与 label；解析失败样本可记为空预测。
2. **过滤类别**：仅保留 9 个目标类别（把 `text` 等非目标类别去掉）。
3. **坐标合法化**：
   - 保证 `x1 < x2, y1 < y2`；
   - 裁剪到图像边界内；
   - 过滤面积过小/非法框。
4. **坐标回映射**（若有 letterbox/pad）：把框映射到评测坐标系。
5. **置信度处理**：
   - 若模型没有显式 score，可采用统一分数（如 1.0）或从生成信息中构造置信度；
   - 评测工具通常需要 `(bbox, class, score)`。

### 2.5 评测：mAP50

你要求指标是 **mAP50**，即 IoU 阈值固定为 0.5 的 mean AP。

- 对每个类别分别计算 AP@0.5；
- 再对 9 个目标类别取平均得到 mAP50。

可写成：

\[
\text{mAP50} = \frac{1}{9}\sum_{c\in C} AP_c(\text{IoU}=0.5)
\]

其中 \(C\) 为上述 9 类集合。

**口径一致性要求**（最容易出错的地方）：

- GT 只保留 9 类；
- 预测也只保留 9 类；
- AP 只在这 9 类上算平均。

---

## 3. 输出（Output）

输出分两层：样本级输出和数据集级输出。

### 3.1 样本级输出

每页推理后，保存结构化检测结果，例如：

```json
[
  {"bbox": [x1, y1, x2, y2], "label": "title", "score": 1.0},
  {"bbox": [x1, y1, x2, y2], "label": "figure", "score": 1.0}
]
```

可选地附带：

- 原始模型文本输出（用于排错）
- JSON 解析状态（成功/失败）
- 预处理与回映射参数

### 3.2 数据集级输出

至少应汇总：

1. **总体 mAP50（主指标）**
2. **各类别 AP50 明细**（title/abstract/.../math）
3. 样本统计：
   - 总页数（1000）
   - 有效预测页数
   - 空预测页数
   - JSON 解析失败页数（若存在）

推荐输出格式：

- 控制台打印（快速查看）
- `metrics.json`（便于复现实验与对比）

---

## 4. 一句话总结流程

**输入**：Qwen2.5VL 3B checkpoint + 1K 测试页图像 + 过滤后的 9 类 GT + 固定 prompt。  
**处理**：预处理到目标尺寸 → 模型生成 JSON 检测结果 → 解析/过滤/坐标校正 → 按 9 类计算 AP@0.5。  
**输出**：每页检测框结果与全数据集 `mAP50`（含各类 AP50 细项）。
